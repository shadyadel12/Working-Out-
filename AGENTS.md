# Project Working Notes

## User-facing changelog

Whenever a task adds a feature, changes app behavior, improves security, or makes another meaningful user-visible improvement, update `CHANGELOG.md` before finishing.

- Date every update and place the newest update first.
- Write for non-technical app users in short, plain-English sentences.
- Use only applicable headings: New Features, Improvements, Fixes, and Security.
- Explain what users notice or can do; avoid code terms, file names, implementation details, and technical jargon.
- For security updates, describe the protection positively without explaining weaknesses or attack methods.
